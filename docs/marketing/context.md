# 마케팅 컨텍스트 기초 — M0 Phase

**초안 작성:** 2026-04-18  
**상태:** M0 진행 중

---

## 1. 제품 정의

### 한줄 정의
**Tourwave**는 한국 레저 스포츠(패러글라이딩, 스쿠버 다이빙, 서핑, 카약, ATV 등) 투어/액티비티 운영사를 위한 **B2B 예약 관리 플랫폼**으로, 운영사의 고객 예약·운영 실행·사후 관리(출석, 리뷰, 환불)를 통합하여 관리하고, B2C 소비자에게는 **신뢰성 높은 예약·결제 경험**을 제공한다.

### 핵심 기능 영역
- **예약 관리:** 수용 인원(정원)과 실시간 좌석 할당, 대기열 관리, 예약 상태 추적
- **결제/환불:** 안전한 결제 통합, 명확한 환불 정책, 자동 환불 처리
- **운영 실행:** 일정 관리, 참가자 확인, 취소/변경 처리, 감사 이벤트 기록
- **사후 관리:** 참가 확인, 고객 리뷰, 분쟁 해결

## 2. 타겟 시장

### 시장 규모 추정 (한국 레저 스포츠)

**한국 레저 스포츠 시장 배경:**
- 국내 해수욕장 방문자: 연 ~1억 명 (스포츠 레저 활동 포함)
- 패러글라이딩 동호인: ~30,000-50,000명, 주요 지역 제주·강원·남해
- 스쿠버 다이빙: 연 500-600만 명의 해양 레저 참가, 전문 다이버 ~40,000명
- 서핑/카약: 연간 성장률 15-20%, 특히 MZ 세대와 원주민 관광객 중심

**북극성 지표(NSM) 후보:**
1. **운영사 월 활성 사용자(MAU):** 초기 목표 50개 → Year 1 500개
2. **월 예약 거래액(GMV):** 초기 3억 원 → Year 2 100억 원
3. **소비자 리피트율:** 초기 15% → Year 1 40% (재예약율)

### 지역 전개 순서
1. **Phase 1 (Month 1-3):** 제주 (패러글라이딩, 스쿠버 다이빙 집중)
2. **Phase 2 (Month 4-6):** 강원 (카약, ATV), 경주 (해양 액티비티)
3. **Phase 3 (Month 7-12):** 전국 확산

## 3. 문제 정의

### 운영사 관점의 Pain Points

#### Pain 1: 수동 예약 관리의 비효율
- **현황:** 전화, 카카오톡, 구글폼, 엑셀로 분산 관리 → 중복 예약, 정원 초과 사건
- **비용:** 소규모 운영사(1-3인)는 월 100-300건 예약을 수동으로 처리 → 주 15-20시간 소비
- **기회 손실:** 실시간 좌석 조회 불가 → 예약 한 건당 확인 전화 필요

#### Pain 2: 결제·환불 프로세스 복잡도
- **현황:** 카카오페이/계좌이체 수동 입금 확인 → 미수금 발생, 환불 분쟁
- **신뢰도:** 고객이 "정말 예약 되었나?" 불안감 → 재확인 전화 증가
- **컴플라이언스:** 구두 약속과 기록 불일치 → 소비자 분쟁 심화

#### Pain 3: 데이터 추적 및 의사결정의 부족
- **현황:** 월 매출, 취소율, 고객 만족도를 파악하기 어려움
- **운영:** 어느 시간대가 인기 있는지, 취소 패턴을 모름 → 스케줄 계획 미흡
- **마케팅:** 고객 재방문 데이터 부재 → 타게팅 마케팅 불가능

### 소비자 관점의 Pain Points

#### Pain 1: 예약 채널의 분산과 신뢰 부족
- **현황:** 네이버 검색 → 개별 업체 홈페이지/카카오톡 → 전화 예약 → 입금 대기
- **불신:** 운영사마다 다른 규칙(환불 정책, 취소 수수료) → 예약 후 불만
- **선택 기준:** 리뷰 부족 → 첫 이용 고객이 선택하기 어려움

#### Pain 2: 결제·환불 불안감
- **현황:** 선입금 후 취소 불가, 또는 "환불은 2주 후" 같은 모호한 약속
- **신뢰도:** 결제 기록이 문자/음성만 있음 → 분쟁 시 증거 부족
- **편의성:** 카카오페이나 계좌이체 수동 입금만 가능 → 신용카드, 국제 결제 불가

#### Pain 3: 사후 경험 단절
- **현황:** 예약 후 → 당일 연락 대기 → 활동 후 관리 없음
- **리뷰 부재:** 만족했지만 리뷰 남길 곳이 없음 → 입소문 효과 제한
- **취소·변경:** 취소하고 싶어도 "누구한테 연락?" → 고객 여정 끝남

## 4. 솔루션 요약

### Tourwave의 차별화 요소

1. **통합 예약 관리 (B2B):** 
   - 운영사는 Tourwave 대시보드 하나에서 모든 예약을 실시간 추적
   - 정원 자동 관리, 중복 예약 제거, 자동 대기열 처리
   - 라이브 캘린더 + API 연동으로 타 채널(OTA) 싱크 지원

2. **안전한 결제·환불 (B2B + B2C):**
   - PG사(결제 게이트웨이) 통합 → 투명한 기록
   - 정책 기반 자동 환불: 취소 기한 내 100%, 이후 일부 환불 규칙 자동화
   - 소비자에게는 선명한 취소 정책 명시

3. **신뢰성 높은 커뮤니티 (B2C):**
   - 공식 플랫폼 → 개별 카카오톡/전화 대신 구조화된 소통
   - 운영사 프로필 + 리뷰 시스템 → 첫 예약 고객도 신뢰 가능
   - 결제 확인 알림, 환불 상태 실시간 추적

4. **운영 효율화 (B2B):**
   - 대시보드: 월 GMV, 취소율, 고객 만족도 지표 한눈에
   - 자동화: 결제 확인, 환불 처리, 대기열 관리
   - API: 기존 채널(야놀자, 네이버 관광) 자동 싱크 가능

## 5. 핵심 가치 제안

### 운영사(B2B) 가치
1. **월 10-15시간 시간 절약** → 수동 예약 관리 자동화
2. **환불·분쟁 감소 50%** → 명확한 정책 + 자동 처리
3. **재방문율 +20% 추정** → 데이터 기반 마케팅 + 고객 추적

### 소비자(B2C) 가치
1. **일관된 예약 경험** → 모든 액티비티가 같은 방식으로 예약·취소
2. **결제 신뢰도** → PG사 인증 + 실시간 환불 추적
3. **커뮤니티 접근성** → 리뷰 기반 선택, 투명한 취소 정책

## 6. 경쟁 환경 및 포지셔닝

### 경쟁자 맵
| 플랫폼 | 타겟 | 강점 | 약점 |
|--------|------|------|------|
| **야놀자** | 숙박+액티비티 | 유명도, 결제 인프라 | 액티비티 관리 미흡, 운영사 수수료 높음(10-15%) |
| **마이리얼트립** | 여행객 + 투어 | 해외 확장, UX | 운영사 온보딩 느림, 로컬 레저는 약함 |
| **개별 업체** | 로컬 고객 | 직접 관계, 높은 마진 | 예약 시스템 없음, 마케팅 불가, 신뢰도 낮음 |

### Tourwave의 포지셔닝
> **"레저 스포츠 운영사를 위한 #1 예약 관리 플랫폼"**
> 
> 야놀자보다 **낮은 수수료(3-5%)** + **액티비티 특화 기능**  
> 마이리얼트립보다 **빠른 온보딩** + **로컬 마케팅 연계**

## 7. 초기 성공 정의 (Proof of Concept)

### M0 종료 기준
- [ ] 운영사 5-10개사 온보딩 (제주)
- [ ] 월 예약 거래액 3,000-5,000만 원
- [ ] 소비자 리뷰 50개+ (평점 4.5/5.0+)
- [ ] 환불율 5% 이하 (정상 범위)

### M1-M3 마일스톤
- **M1 (4월-5월):** 운영사 50개 → GMV 3억 원
- **M2 (6월-8월):** 강원 지역 확대 → 운영사 200개 → GMV 10억 원
- **M3 (9월-12월):** 전국 확산 → 운영사 500개+ → GMV 50억 원
