# feat: UserStatus enum 완성 (T-005)

## 배경 및 목표

`UserStatus` enum에 `SUSPENDED`, `DELETED` 가 이미 선언되어 있으나 실제 전이 로직이 없고,
소프트 딜리트(`deleted_at_utc` 컬럼) 및 개인정보 마스킹이 미구현된 상태.
4개 상태의 전이 규칙을 도메인 엔티티에 명확히 정의하고,
`UserCommandService`에 `suspend()`, `delete()`, `restore()` 메서드를 추가한다.

## Impact / Rollback / Verification (에스컬레이션 승인됨)

- **Impact**: `users` 테이블에 `deleted_at_utc DATETIME(6) NULL` 컬럼 추가 — nullable이므로 기존 데이터·쿼리 무중단
- **Rollback**: 마이그레이션 파일 삭제 + `ALTER TABLE users DROP COLUMN deleted_at_utc`
- **Verification**: `SHOW COLUMNS FROM users LIKE 'deleted_at_utc'` 로 컬럼 존재 확인
