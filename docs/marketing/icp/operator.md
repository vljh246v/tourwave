# 운영사 ICP (B2B) — M0 Phase

**초안 작성:** 2026-04-18  
**상태:** 3개 페르소나 정의 완료

---

## 개요
Tourwave의 B2B 타겟 고객은 월 50-500건의 예약을 처리하는 레저 스포츠 투어/액티비티 운영사다. 초기 타겟은 **제주 기반 소규모 운영사(Persona A)**부터 중형 리조트까지 포함한다.

---

## Persona A: 패러글라이딩 스쿨 운영사 (소규모 1-3인)

### 기본 정보
| 항목 | 내용 |
|------|------|
| **이름** | 최준호 (39세) |
| **직책** | 스쿨 대표 겸 강사 |
| **회사명** | 제주스카이항공스포츠 (가상) |
| **회사 규모** | 1인 운영 + 임차 강사 3-5명 |
| **지역** | 제주 서귀포 중문 해변 |
| **연매출** | ~2억 원 (월 15-20건 × 가격 80-120만 원) |

### 배경
- 10년간 패러글라이딩 강사 경력, 5년 전 독립하여 학원 창립
- 초기에는 입소문으로만 운영했지만, 갈수록 경쟁이 심화
- 현재 고객 50%는 개인 고객, 50%는 숙박 업체 추천이나 여행사 알선
- 소셜 미디어(인스타그램)에 자체 콘텐츠를 올리지만, 예약 채널은 여전히 전화/카톡

### 현재 예약 관리 방식
```
고객 문의 → 카카오톡/전화 응답 → 구글폼 예약 → 엑셀 수동 입력 
→ 계좌이체/카카오페이 확인 (수동) → 당일 문자 알림 (수동)
```

**문제점:**
- 월 15-20건 예약을 처리하는 데 주 5-8시간 소비
- 중복 예약 또는 취소를 깜빡했던 사건이 월 1-2회
- 환불 요청 시 "계좌 확인했나?" 재확인 전화 필요
- 날씨 악화로 당일 취소가 많은데, 고객에게 그때그때 연락해야 함

### Pain Points (상세)
1. **예약 관리의 혼란:** 카톡 몇 건, 구글폼 몇 건 → 누가 확인됐는지 헷갈림 → 당일 "예약 확인 안 돼" 전화 자주 옴
2. **결제 추적 미흡:** 계좌이체는 누군지 확인이 오래 걸리고, 매월 환불액 계산이 번거로움
3. **마케팅 데이터 부재:** 성수기/비수기 정도는 느끼지만, 고객 나이대, 재방문율 같은 데이터가 없음 → 타게팅 광고 불가능
4. **운영 비용:** 예약 관리만으로 시간이 많이 들어, 고객 서비스나 신규 상품 개발에 집중하기 어려움

### 구매 결정권자
- 본인(대표)가 전적으로 결정
- 월 비용이 100만 원 이상이면 "우선 매출 증가부터"하는 성향

### 주로 사용하는 채널
- **예약 문의:** 카카오톡(70%), 전화(20%), 인스타 DM(10%)
- **정보 수집:** 유튜브(강사 영상), 인스타그램(고객 후기), 동료 소개
- **의사결정:** 다른 운영사 추천, 월간 비용 ROI, "쉬워야 함"

### 요구 사항 (Wishlist)
- "대시보드에서 한눈에 이번 주 예약 몇 개인지 보고 싶어"
- "환불을 자동으로 처리해 주는 시스템이 있으면 좋겠어"
- "고객이 내 서비스를 어디서 찾았는지 알고 싶어"
- **가격:** 월 15-20만 원 선호 (현재 광고비 50만 원 대비 합리적)

## Persona B: 스쿠버 다이빙 리조트 (중형 5-15인)

### 기본 정보
| 항목 | 내용 |
|------|------|
| **이름** | 이지영 (45세) |
| **직책** | 리조트 운영 매니저 |
| **회사명** | 제주디프블루다이브 (가상) |
| **회사 규모** | 5-15명 (강사 7명, 사무/마케팅 2명, 지원 3명) |
| **지역** | 제주 동쪽 성산/하도 해변 |
| **연매출** | ~8억 원 (월 150-200건 × 평균 40만 원) |

### 배경
- 스쿠버 다이빙 경력 15년, 리조트 운영 8년
- 한국 스쿠버 동호인의 선호 장소 TOP 3에 꼽히는 시설
- 고객층: 자격증 취득 입문자(30%), 경험자 숙련자(70%)
- 해외 투어객 유입도 많아서, 영문 페이지와 다국어 지원이 중요
- 현재 예약은 야놀자(40%), 개별 전화(30%), 마이리얼트립(20%), 기타(10%)

### 현재 예약 관리 방식
```
야놀자/마이리얼트립 → 자체 시스템(구글시트) 동기화 (수동)
→ 강사 일정 조율 (카톡) → 당일 가이드 배정 (수동)
→ 결제 확인 (야놀자 대시보드 확인)
```

**문제점:**
- 야놀자는 수수료 12%, 마이리얼트립은 10% → 월 1,000만 원 이상 수수료 비용
- 여러 채널에서 동시에 예약이 들어오면 정원이 초과되는 경우 발생
- 강사 일정 조율이 카톡으로만 되어 있어서, 공식 기록이 없음 → 소비자 분쟁 시 증거 부족
- 환불 정책이 명확하지 않아서, 고객마다 다르게 처리되는 경향

### Pain Points (상세)
1. **높은 중개 수수료:** 야놀자·마이리얼트립 통합하면 월 1,500만 원 이상의 수수료 → "이들을 빠져나갈 방법이 있을까?"
2. **채널 관리의 복잡성:** 3개 채널을 동시에 모니터링 → 예약 중복이나 캔슬 누락 위험
3. **강사 스케줄 관리:** 강사별로 자격증 등급이 다르고, 선호하는 다이빙 사이트가 다른데, 이를 자동으로 배정하기 어려움
4. **국제 결제:** 해외 고객 결제가 복잡하고, 환율 변동 헤징 없음 → 매출 예측이 어려움

### 구매 결정권자
- 운영 매니저(본인) + 대표 승인 필요 (매월 200만 원 이상 투자 시)
- 강사 대표와의 협의도 중요 (스케줄 영향)

### 주로 사용하는 채널
- **의사결정:** ROI(수수료 대비 매출 증가), 사용 편이성, 고객 평점 영향
- **정보 수집:** 업계 커뮤니티(다이버 포럼), 다른 리조트 벤치마킹, 전시회(관광 엑스포)
- **도입:** "다른 리조트가 쓰고 있는가?", "AS와 기술 지원이 충분한가?"

### 요구 사항 (Wishlist)
- "여러 채널의 예약을 자동으로 한 곳에 통합하고 싶어"
- "강사 일정을 자동으로 최적화해 주는 기능이 있으면"
- "환불 정책을 명확히 정의하고, 그에 따라 자동 처리되는 시스템"
- "월 매출, 취소율, 고객 만족도를 대시보드에서 한눈에 보고 싶어"
- **가격:** 수수료 형식(GMV 3-5%) 또는 월 정액(300-500만 원) → 높은 수수료보다는 자체 플랫폼 이탈 목표

## Persona C: 서핑/카약 시즌 비즈니스 (부 수입/계절 운영)

### 기본 정보
| 항목 | 내용 |
|------|------|
| **이름** | 박해준 (35세) |
| **직책** | 가이드 겸 주말 운영사 |
| **회사명** | 남해서핑클럽 (가상/개인 사업) |
| **회사 규모** | 1인 운영 (주말 조수 2-3명 임차) |
| **지역** | 경남 남해군 해변 |
| **연매출** | ~6,000만 원 (여름/가을 성수기만 운영, 월평균 500만 원) |

### 배경
- 낮에는 해양 관광 가이드, 주말과 성수기에 서핑/카약 강습 운영
- 고객 대부분 여행객 → 당일 예약이 많고, 재방문율은 낮음
- 수익성은 높지만(마진율 70%), 계절성 때문에 월수입이 불안정
- 현재 예약: 에어비앤비(일부 숙박 고객), 개인 전화, 지인 소개

### 현재 예약 관리 방식
```
전화 요청 → 날씨·파도 조건 확인 (개인 경험) → 카톡으로 확정 
→ 당일 현장 만남 → 현금 수령 또는 계좌이체 후 정산
```

**문제점:**
- 당일 취소가 많은데 (날씨), 이미 조수 비용은 발생 → 손실 발생
- 예약 기록이 카톡뿐 → 나중에 분쟁되면 증거가 모호함
- 여행객들이 "정말 내일 가능한가?" 여러 번 물어봄 → 응답 부담

### Pain Points (상세)
1. **계절성 불안정:** 겨울에 수입이 거의 없어서, 연간 마진을 높이려고 더 많은 예약을 받으려 하지만, 관리 체계가 없어서 오버부킹 위험
2. **당일 취소 손실:** 날씨 악화로 당일 취소 시, 고객은 환불받지만 본인은 조수 비용을 감당 → 불공정성 느낌
3. **고객 신뢰도:** 예약 기록이 없어서 고객이 "진짜 예약 되었나?" 불신 → 전날 밤 재확인 전화
4. **마케팅 기회 상실:** 만족한 고객들이 많지만, 그들이 추천할 수 있는 공식 채널이 없음 → 입소문만 의존

### 구매 결정권자
- 본인이 100% 결정
- 월 비용보다는 "추가 매출이 생기는가?"에 민감

### 주로 사용하는 채널
- **정보 수집:** 유튜브(레저 트렌드), 지인 소개, 업계 카톡방
- **의사결정:** "간단해야 함" (복잡한 시스템 학습 시간 없음), 즉시 현금화, 추가 수입

### 요구 사항 (Wishlist)
- "예약을 받으면 자동으로 기록되고, 고객도 확인할 수 있게"
- "당일 취소는 환불 규칙을 명확히 (예: 2시간 전까지 전액 환불)"
- "리뷰가 쌓여서, 새로운 고객들이 찾아오는 효과"
- **가격:** 건당 2-3% 수수료 또는 월 5-10만 원 (낮은 기준)

## 운영사 ICP 타입 분류표

| 구분 | Persona A (패러글라이딩) | Persona B (스쿠버) | Persona C (서핑/카약) |
|------|---------|---------|---------|
| **규모** | 1-3인 | 5-15인 | 1-3인 (부업) |
| **월 예약** | 15-20건 | 150-200건 | 20-30건 (계절) |
| **연매출** | ~2억 원 | ~8억 원 | ~6,000만 원 |
| **주요 Pain** | 수동 관리 시간 | 수수료 부담 + 채널 관리 | 계절성 + 신뢰도 |
| **의사결정 기준** | ROI (시간 절약) | 수수료 절감 + 효율화 | 간편성 + 추가 수입 |
| **초기 가격 민감도** | 중간 (월 15-25만 원) | 낮음 (수수료형) | 낮음 (건당 2-3%) |
| **온보딩 난이도** | 낮음 | 중간 | 낮음 |

## 우선 순위 및 온보딩 순서

### Phase 1 타겟 (Month 1-2)
**Persona A + C 집중:**
- 1-3인 소규모 운영사 5-10개사
- 온보딩이 빠르고, 입소문 효과가 높음 (소규모 커뮤니티)
- 월 비용 기준으로 합의하기 쉬움

### Phase 2 타겟 (Month 3-4)
**Persona B 진출:**
- 중형 리조트/스쿨 5개사
- 더 높은 GMV 생성 → 플랫폼 안정성 증명 후
- 수수료 또는 정액 모델 협상
