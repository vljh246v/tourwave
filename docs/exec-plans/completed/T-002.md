# T-002 — Refresh token 로테이션 race condition 해결

## 배경 및 목표

`AuthTokenLifecycleService.rotate()`가 revoke → issue 순서로 구현되어, 동시 요청 시 race condition이 발생한다.
JPA Optimistic Lock(@Version)을 사용해 동시 로테이션 시 먼저 온 요청만 성공하고, 나머지는 409 REFRESH_TOKEN_ROTATION_CONFLICT를 반환하도록 한다.

## 사용자 결정 사항 (2026-04-26)

- **Lock 전략**: JPA Optimistic Lock (@Version 필드) 단독 사용
- **SELECT ... FOR UPDATE 사용 금지** (카드 §구현 지침 #3의 FOR UPDATE 문구 무시)
- **도메인 예외**: OptimisticLockException → REFRESH_TOKEN_ROTATION_CONFLICT → HTTP 409
- **마이그레이션**: `V<ts>__add_refresh_token_version.sql` — `version BIGINT NOT NULL DEFAULT 0`
