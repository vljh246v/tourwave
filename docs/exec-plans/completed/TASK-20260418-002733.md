# chore: bootJar mainClass 명시적 설정으로 두 진입점 분리 (TASK-20260418-002733)

## 배경 및 목표

같은 Gradle 모듈에 `TourwaveApplication`과 `WorkerApplication` 두 개의
`@SpringBootApplication` 진입점이 존재하여 `bootJar`/`bootRun` 태스크 실행 시
`Unable to find a single main class` 오류로 빌드가 실패하는 문제를 수정한다.

- `bootJar` / `bootRun` → `TourwaveApplication` (API 서버, 기본)
- `bootJarWorker` (신규 태스크) → `WorkerApplication` (배치 워커)

## 구현 단계

- [ ] 1. `build.gradle.kts`에 `tasks.bootJar { mainClass.set(...) }` 추가
- [ ] 2. `tasks.bootRun { mainClass.set(...) }` 추가
- [ ] 3. `tasks.register<BootJar>("bootJarWorker")` 신규 태스크 등록
- [ ] 4. `./gradlew bootJar` 빌드 성공 확인
- [ ] 5. `verify-task.sh`로 검증

## 완료 기준

- [ ] `./gradlew bootJar` 성공 (API 서버 jar)
- [ ] `./gradlew bootJarWorker` 성공 (워커 jar)
- [ ] 검증기 5개 전부 통과
