- 같은 패턴 **3회 반복** → 린트/테스트로 **구조적 불가능화** 필수

## 기록 방법

- **ID**: `F-YYYY-MM-DD-N`
- **증상**: 외부에서 관찰 가능한 현상
- **근본 원인**: 구조적 사유
- **강화책**: ADR, 린트, 검증기, 문서 링크
- **도메인**: 워크플로우 / 보안 / 아키텍처 / 테스트 / 문서 등
