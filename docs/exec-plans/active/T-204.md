# T-204 — Asset Content-Type 화이트리스트 검증

## 결정 기록

- 422 매핑: GlobalExceptionHandler가 DomainException.status 그대로 사용하는 데이터-드리븐 패턴 확인. adapter/in/web 수정 없이 ErrorCode enum 추가만으로 충분.
- docs/policies.md 화이트리스트 반영: Phase 7.6에서 orchestrator 처리 (tdd-backend 스코프 제외).
- requireContentType() 삭제: AssetContentType.fromString()이 blank 검사 포함 — 기능적으로 동등하며 더 엄격.
- 대소문자 정규화: fromString()에서 trim().lowercase() 처리 — "IMAGE/JPEG" → 정상 처리.
