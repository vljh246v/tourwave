# 소비자 ICP (B2C) — M0 Phase

**초안 작성:** 2026-04-18  
**상태:** 3개 페르소나 정의 완료

---

## 개요
Tourwave의 B2C 타겟 고객은 레저 스포츠 액티비티를 경험하려는 여행객 및 로컬 주민이다. 초기 타겟은 **20-30대 개별 여행객(Persona A)**부터 가족 단위, 동호회까지 포괄한다.

---

## Persona A: MZ 개별 여행객 (독립 여행, 새로운 경험 추구)

### 기본 정보
| 항목 | 내용 |
|------|------|
| **이름** | 김수연 (28세) |
| **직업** | 마케팅 회사 사원 |
| **소득** | 연 5,000만 원 (세후 월 350만 원) |
| **거주** | 서울 강남구 |
| **성향** | 개별 여행 선호, SNS 활동 활발, 경험에 투자 |

### 여행 습관
- **여행 빈도:** 연 4-6회 (휴가 때 + 장기 주말 활용)
- **여행 형태:** 주로 혼자 또는 친구 2-3명 그룹
- **선호 지역:** 제주, 부산, 강원 해변
- **활동 관심:** 새로운 경험(액티비티) > 관광지 둘러보기
- **예산:** 1회 여행당 50-100만 원 (숙박 + 액티비티)

### 예약 여정 (현재)

```
1. 정보 수집
   → 네이버 검색 ("제주 패러글라이딩") 
   → 인스타그램 (#제주_액티비티 해시태그) 
   → 유튜브 (액티비티 후기 영상)

2. 비교 및 평가
   → 개별 업체 홈페이지 (대부분 낡은 디자인)
   → 구글 맵 리뷰 (수 적음, 최신 의견 부족)
   → 인스타그램 DM 또는 전화 문의 (응답 시간 오래 걸림)

3. 의사결정
   → 가격 비교 (대부분 비슷함)
   → 리뷰/후기 (정보 부족)
   → "다른 사람들은 이 업체를 선택했나?" (소셜 증명)

4. 예약 및 결제
   → 카카오톡 예약 확인 (기록 불분명)
   → 계좌이체 또는 카카오페이로 선입금 (환불이 불안함)
   → 당일 "정말 예약 돼 있나?" 재확인 전화

5. 활동 및 사후
   → 당일 만남 (강사를 처음 봄)
   → 활동 후 SNS에 사진 올림
   → 리뷰는 남길 곳이 없음 → 입소문만 가능
```

### Current Pain Points (상세)

1. **정보 부족 & 신뢰 부족:**
   - 검색 결과가 분산: 네이버, 인스타, 개별 웹사이트 → 정보 정합성 없음
   - 리뷰가 적거나 구식: 대부분 1-2년 전 리뷰 → 현재 상태 알 수 없음
   - "정말 이 업체가 안전한가?" 불확실성 → 입장료를 날릴 수도 있다는 불안감

2. **예약 과정의 마찰:**
   - 카카오톡 예약 → 응답이 늦거나 생략될 수 있음 (업체 사정)
   - 예약 확인이 문자/카톡으로만 기록 → 분쟁 시 증거 부족
   - "예약이 정말 들어가 있나?" 재확인을 위해 당일 아침에 전화해야 함

3. **결제 불안감:**
   - 선입금 방식만 가능 → 취소 시 "환불을 언제까지 기다려야 하나?"
   - 환불 정책이 명확하지 않음: "7일 이상 전 취소만 환불" vs "2일 전까지 환불"
   - 계좌이체 확인이 오래 걸림 → "입금이 도착했나?" 다시 물어봐야 함

4. **취소/변경 어려움:**
   - 취소하려면 "누구한테 연락?" → 카톡이 답답하고, 전화도 깨지거나 응답 없음
   - 예약 변경(날짜 연기, 인원 변경)도 이메일/카톡으로만 가능
   - 취소 수수료가 명확하지 않아서, "정말 전액 환불인가?" 불신

### 예약 결정 요인 (우선순위)
1. **리뷰 & 평점** (40%) — "다른 사람들이 만족했는가?"
2. **안전성** (25%) — 강사 자격, 시설, 날씨 안전 정책
3. **가격** (20%) — 합리적 수준 (극저가보다는 신뢰도)
4. **접근성** (15%) — 예약/결제의 편의성

### 사용 채널 (정보 수집)
| 채널 | 사용율 | 용도 |
|------|--------|------|
| 네이버 검색 | 70% | 초기 정보 수집, 리뷰 찾기 |
| 인스타그램 | 60% | 시각적 후기, 업체 팔로우 |
| 유튜브 | 50% | 장시간 후기 영상 시청 |
| 구글 맵 | 40% | 위치, 평점, 사진 |
| 에어비앤비 | 30% | 액티비티 패키지 확인 |
| 카카오톡 | 80% | 예약 확인 (이미 결정한 후) |

### Tourwave에 바라는 것 (Wishlist)
- "한 곳에서 모든 액티비티를 검색하고 비교하고 싶어" (네이버, 인스타, 개별 사이트 다니고 싶지 않음)
- "리뷰가 충분해야 결정하는데, 이전 고객들의 리뷰를 보고 싶어"
- "예약 후 환불 정책을 명확히 알고 싶고, 취소하기 쉬웠으면"
- "결제가 안전한 채널(신용카드, 페이)이었으면 좋겠어"
- "예약 상태를 실시간으로 추적할 수 있게"
- **예약 가격:** 같은 서비스라면 Tourwave 통해서도 상관없음 (동일 가격 보장)

## Persona B: 30-40대 가족 여행객 (자녀 동반, 안전성 우선)

### 기본 정보
| 항목 | 내용 |
|------|------|
| **이름** | 이준석 (42세) + 아내 박미라 (40세) |
| **직업** | IT 회사 부장 + 가사/부업 |
| **소득** | 연 1.2억 원 (세후 월 700만 원) |
| **자녀** | 초등 4학년 + 6학년 |
| **거주** | 서울 서초구 |
| **성향** | 가족 경험 투자 우선, 안전성 민감, 리뷰 의존 |

### 여행 습관
- **여행 빈도:** 연 2-3회 (방학 때 주로)
- **여행 형태:** 가족 (4-5명)
- **선호 지역:** 제주, 강원 (아이 경험용)
- **활동 관심:** 아이가 안전하게 배울 수 있는 활동
- **예산:** 1회 여행당 200-400만 원 (숙박 + 액티비티 + 식사)
- **숙박 선호:** 리조트 또는 펜션 (편의 시설 중요)

### 예약 여정 (현재)

```
1. 정보 수집
   → 야놀자/마이리얼트립 (패키지 상품 검색)
   → 블로그 (육아 블로거의 후기 찾기)
   → 네이버 카페 (육아/여행 정보)

2. 안전성 검증
   → "아이 참여 가능한가?" 확인
   → 강사 자격, 안전 기록 검색
   → 마이너스 리뷰 확인 (사고 등)

3. 의사결정
   → "다른 부모들이 추천하는가?"
   → 가격 (너무 싼 건 의심)
   → 패키지 포함 항목 (식사, 이동 등)

4. 예약
   → 야놀자/마이리얼트립 통한 예약 (신용카드, 안전)
   → 추가 요청사항은 전화로 (예: 아이 이용 시 더 작은 장비)

5. 활동 및 사후
   → 당일 강사 만남
   → 안전 교육 후 활동
   → 사진/영상 촬영
   → 블로그에 후기 작성 (다른 가족에 추천)
```

### Pain Points (상세)

1. **패키지화된 옵션 부족:**
   - 개별 액티비티만 있고, 숙박+액티비티 패키지가 적음
   - 야놀자/마이리얼트립은 일부만 포함 → 다른 액티비티는 직접 찾아야 함

2. **안전성 정보 부재:**
   - 강사의 자격증, 경력, 안전 기록을 알 수 없음
   - 아이 이용 시 특수 요청(장비 크기 조정, 나이별 커리큘럼)이 명확하지 않음
   - "혹시 사고난 기록은 없나?" → 검증 방법이 없음

3. **의사결정에 시간 많이 소요:**
   - 여러 채널 검색하며 비교 (야놀자, 블로그, 직접 업체 문의)
   - "이 업체가 정말 믿을 수 있는가?" 확인에 며칠 소요

4. **예약 후 커뮤니케이션:**
   - 야놀자에서 예약했지만, 추가 요청(아이 용품 준비)은 직접 전화해야 함
   - "당일 날씨는 괜찮나?" "혹시 취소되는 건 아닌가?" 불안감 → 당일 아침 재확인 전화

### 예약 결정 요인 (우선순위)
1. **안전성** (50%) — 강사 자격, 안전 기록, 리뷰
2. **패키지 구성** (20%) — 아이 맞춤형 옵션 포함 여부
3. **리뷰 신뢰도** (20%) — 다른 부모의 후기
4. **가격** (10%) — 합리적 수준 (안전 > 가격)

### 사용 채널 (정보 수집)
| 채널 | 사용율 | 용도 |
|------|--------|------|
| 야놀자/마이리얼트립 | 80% | 패키지 상품, 예약, 결제 |
| 블로그 | 60% | 육아 블로거의 솔직 후기 |
| 네이버 카페 (육아) | 50% | 다른 부모들의 추천 |
| 구글 맵 | 40% | 위치, 리뷰 |
| 인스타그램 | 30% | 시각적 정보 (자녀와 함께) |

### Tourwave에 바라는 것 (Wishlist)
- "아이가 참여 가능한 액티비티를 명확히 구분해 줬으면" (나이, 경험 요구도)
- "강사의 자격증, 경력, 안전 기록을 한눈에 볼 수 있게"
- "아이와 함께한 다른 부모들의 리뷰를 보고 싶어"
- "패키지로 여러 액티비티를 묶어서 예약할 수 있게"
- "날씨 취소 정책을 명확히" (환불 vs 일정 연기)
- **예약 가격:** 야놀자/마이리얼트립과 비슷하면 됨 (단, 안전성과 리뷰가 우수해야 함)

## Persona C: 동호회/단체 예약 (10인+, 그룹 할인 기대)

### 기본 정보
| 항목 | 내용 |
|------|------|
| **이름** | 최현진 (38세) |
| **역할** | 패러글라이딩 동호회 회장 |
| **회원 규모** | 30-50명 (월 활동 5-10명) |
| **연령대** | 30-50대 취미 애호가 |
| **거주** | 서울 및 수도권 (활동지: 제주, 강원) |
| **성향** | 단체 활동 조직 경험 많음, 가성비 민감, 관계 우선 |

### 단체 활동 형태
- **월 활동:** 2-4회 (주로 주말)
- **인원:** 회차당 10-20명 (그룹 할인 가능 규모)
- **목표:** 취미 공유 + 친목 + 경험 확대
- **예산:** 회원당 10-50만 원 (비용 공동 부담)

### 예약 여정 (현재)

```
1. 정보 수집
   → 회원들의 입소문 ("어디 좋은 데 있어?")
   → 동호회 카톡방 제안 (회원 경험담)
   → 개별 강사와의 관계 (이전에 다녀본 곳)

2. 협상
   → 회장이 직접 업체에 전화 (단체 할인 요청)
   → "10명 예약하면 얼마인가?" 가격 협상
   → 일정 조율 (모두가 가능한 날짜 찾기)

3. 예약
   → 일부는 예약, 일부는 "구두 약속" (나중에 확인)
   → 비용 정산 (회장이 선입금, 나중에 회원에게 징수)

4. 활동
   → 당일 모임
   → 활동 후 회원들이 각자 SNS에 올림
   → 동호회 카톡방에서 후기 공유

5. 사후
   → 단체 사진/영상 공유
   → "다음에는 어디 갈까?" 논의 시작
```

### Pain Points (상세)

1. **단체 할인 협상의 번거로움:**
   - 회장이 직접 여러 업체에 전화해서 가격 비교 → 시간 소비
   - 업체마다 할인율이 다르고, 최소 인원도 다름 (5명? 10명?)
   - 협상이 끝나도 "이 가격이 정말 최저인가?" 의심

2. **인원 확정의 어려움:**
   - 처음에 "10명" 예약했지만, 당일 8명만 나타남 → 비용 정산 문제
   - 아니면 당일 "1명 더 들어가도 되나?" → 업체 입장에선 갑작스러움

3. **비용 정산의 복잡성:**
   - 회장이 선입금하고, 나중에 회원들에게 징수 → 부정확한 기록
   - 누가 얼마를 냈는지 추적이 어려움
   - "환불되면 언제 다시 나눠?" → 복잡한 정산 프로세스

4. **예약 기록의 부재:**
   - 대부분 "구두 약속" → 당일 확인 전화 필수
   - 취소나 변경이 생기면, 누가 통보했는지 불명확

### 의사결정 요인 (우선순위)
1. **가격** (40%) — 단체 할인율, 명확한 비용 구조
2. **편리성** (30%) — 예약 과정 간단함, 온라인 결제
3. **리뷰** (20%) — "다른 단체들도 가는가?"
4. **유연성** (10%) — 인원 변경 용이, 취소 정책 관대

### 사용 채널
| 채널 | 사용율 | 용도 |
|------|--------|------|
| 입소문/회원 제안 | 70% | 초기 정보 |
| 동호회 카톡방 | 90% | 의사결정, 비용 정산 |
| 네이버 검색 | 40% | 추가 옵션 찾기 |
| 인스타그램 | 30% | 시각적 후기 |

### Tourwave에 바라는 것 (Wishlist)
- "단체 예약 시 자동으로 할인 적용되고, 가격이 명확하게 나왔으면"
- "예약 인원을 유동적으로 변경할 수 있게 (10명 → 8명)"
- "비용 정산을 자동으로 해 줬으면" (회원별 정산 링크 제공)
- "예약 확인과 취소/변경을 온라인에서 직접 처리하고 싶어"
- "단체 활동 사진/영상을 공유할 수 있는 공간"
- **예약 가격:** 단체 할인 (10명+ 10% 이상, 20명+ 15% 이상), 선입금 비용 없이 온라인 결제 선호

## 소비자 ICP 타입 분류표

| 구분 | Persona A (개별 여행) | Persona B (가족) | Persona C (단체) |
|------|---------|---------|---------|
| **연령대** | 20-30대 | 30-40대 (아이 포함) | 30-50대 |
| **여행 형태** | 개별/친구 | 가족 (4-5명) | 단체 (10-50명) |
| **의사결정 기준** | 리뷰 + 경험 | 안전성 + 패키지 | 가격 + 편의성 |
| **결제 선호** | 신용카드/페이 | 신용카드 (야놀자) | 온라인 계좌이체/페이 |
| **리뷰 의존도** | 높음 | 매우 높음 | 중간 |
| **가격 민감도** | 중간 | 낮음 | 높음 |
| **환불 정책** | 명확하고 싶음 | 명확해야 함 | 유연해야 함 |

## 고객 여정별 개선 기회 (Tourwave)

### Persona A (개별 여행)
| 기존 Pain | Tourwave 솔루션 | 기대 효과 |
|-----------|-----------------|---------|
| 정보 분산 | 통합 플랫폼 검색 | 10분 → 3분 (정보 수집 시간 70% 단축) |
| 리뷰 부족 | 구조화된 리뷰 시스템 | 신뢰도 +40% |
| 예약 불안 | 실시간 예약 추적 | 당일 재확인 전화 0번 |
| 취소 어려움 | 온라인 취소/환불 | 취소 시간 30분 → 3분 |

### Persona B (가족)
| 기존 Pain | Tourwave 솔루션 | 기대 효과 |
|-----------|-----------------|---------|
| 안전성 정보 부재 | 강사 자격/경력 표시 | 신뢰도 +50% |
| 패키지 부족 | 패키지 상품 제공 | 의사결정 시간 단축 50% |
| 추가 요청 어려움 | 온라인 요청 양식 | 당일 통화 부담 감소 |
| 날씨 불안 | 자동 취소/환불 정책 | 명확성 +80% |

### Persona C (단체)
| 기존 Pain | Tourwave 솔루션 | 기대 효과 |
|-----------|-----------------|---------|
| 할인 협상 | 자동 단체 할인 | 협상 시간 0 |
| 인원 변경 어려움 | 유연한 인원 조정 | 변경 시간 2시간 → 1분 |
| 비용 정산 복잡 | 자동 정산 시스템 | 분쟁 0, 정산 시간 단축 |
| 기록 부재 | 명확한 예약 기록 | 당일 확인 전화 0번 |
